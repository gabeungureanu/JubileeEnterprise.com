Deployment Guide

Deploy Jubilee Browser across your organization with enterprise-grade tools and management options.

Documentation / Deployment Guide

Overview

Jubilee Browser is designed for easy deployment across organizations of all sizes. Whether you're deploying to a small church office or a large school network, this guide provides everything you need to roll out Jubilee Browser efficiently and manage it effectively.

This guide covers:

  • System requirements and prerequisites
  • Silent (unattended) installation
  • Group Policy deployment
  • Configuration management
  • Update management
  • Troubleshooting deployment issues

System Requirements

Before deploying Jubilee Browser, ensure your systems meet these minimum requirements:

Component Minimum Requirement Recommended
Operating System Windows 10 (64-bit) Windows 10/11 (64-bit)
Processor 1 GHz dual-core 2 GHz quad-core or higher
Memory 4 GB RAM 8 GB RAM or more
Disk Space 500 MB available 1 GB available
Network Internet connection for updates Broadband connection

Silent Installation

Silent installation allows you to deploy Jubilee Browser without user interaction. This is ideal for automated deployments via scripts, SCCM, Intune, or other management tools.

Basic Silent Install

Run the installer with the /S flag for silent installation:

JubileeBrowser-Setup-8.0.6.msi /S

Silent Install with Custom Directory

Specify an installation directory using /D:

JubileeBrowser-Setup-8.0.6.msi /S /D=C:\Program Files\JubileeBrowser
Note: The /D parameter must be the last parameter on the command line and must not contain quotes, even if the path contains spaces.

Command Line Options

Option Description
/S Silent mode - no UI displayed
/D=path Custom installation directory
/NCRC Skip CRC check (not recommended)

Group Policy Deployment

For Active Directory environments, you can deploy Jubilee Browser using Group Policy. This method provides centralized control over installation and configuration.

Step-by-Step Deployment

1

Prepare the Installation Package

Copy the Jubilee Browser installer to a network share accessible by all target computers (e.g., \\server\software\jubilee\).

2

Create Deployment Script

Create a batch file or PowerShell script for silent installation:

@echo off \\server\software\jubilee\JubileeBrowser-Setup-8.0.6.msi /S exit /b %errorlevel%
3

Create Group Policy Object

Open Group Policy Management Console and create a new GPO. Navigate to Computer Configuration > Policies > Windows Settings > Scripts > Startup and add your deployment script.

4

Link and Apply GPO

Link the GPO to the appropriate OU containing target computers. Computers will install Jubilee Browser on their next restart.

Configuration Management

Jubilee Browser stores configuration in the user's AppData folder. For enterprise deployments, you can pre-configure settings using configuration files.

Configuration File Location

%APPDATA%\Jubilee Browser\settings.json

Common Configuration Options

Setting Type Description
defaultMode string "internet" or "jubileebibles" - default browsing mode
homePage string Default home page URL
autoUpdate boolean Enable/disable automatic updates
contentFiltering boolean Enable/disable content filtering

Example Configuration

{ "defaultMode": "jubileebibles", "homePage": "https://www.jubileeverse.com", "autoUpdate": true, "contentFiltering": true }

Update Management

Jubilee Browser includes an automatic update system that checks for new versions and applies updates seamlessly. For enterprise environments, you can control update behavior.

Automatic Updates

By default, Jubilee Browser checks for updates on startup and periodically while running. Updates are downloaded in the background and applied when the browser restarts.

Disabling Auto-Updates

To manage updates centrally, you can disable automatic updates and deploy new versions through your existing software management tools:

{ "autoUpdate": false }
Recommendation: If disabling auto-updates, establish a regular update schedule to ensure users receive security patches and new features.

Troubleshooting

Installation Fails Silently

If silent installation fails without error messages:

  • Ensure the user has administrative privileges
  • Check Windows Event Viewer for application errors
  • Verify the installer file isn't corrupted (redownload if needed)
  • Ensure sufficient disk space is available

Browser Doesn't Launch After Installation

  • Verify Windows Defender or antivirus isn't blocking the application
  • Check that all Visual C++ redistributables are installed
  • Try running as administrator

Configuration Changes Not Applied

  • Ensure JSON syntax is valid in configuration files
  • Restart the browser after making configuration changes
  • Verify file permissions allow writing to the AppData folder
